# Security Policy

## Supported Versions

Actualmente se da soporte de seguridad a la rama principal (`main`) y a la version mas reciente publicada.

## Reporting a Vulnerability

Si encuentras una vulnerabilidad:

1. No publiques el exploit en issues publicos.
2. Reporta el problema por mensaje privado al mantenedor del repositorio.
3. Incluye:
   - Descripcion tecnica
   - Pasos para reproducir
   - Impacto esperado
   - Evidencia (logs/capturas)

## Response Targets

- Confirmacion inicial: dentro de 72 horas.
- Evaluacion tecnica: dentro de 7 dias.
- Mitigacion/correccion: segun severidad.

## Scope

Esta politica cubre:
- `nmap pro.py`
- Scripts de instalacion (`install.sh`, `install.ps1`)
- Archivos de automatizacion en `.github/workflows`
